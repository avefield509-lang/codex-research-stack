Integrations

这套系统怎么和外部工具配合。

公开仓库里的集成层不是“能连什么就都连上”，而是强调边界清楚： 什么地方负责正式文献，什么地方负责知识沉淀，什么地方负责证据采集。

返回中文首页 查看 GitHub

Zotero

Zotero 被当成正式文献边界，而不是一个泛化资料仓。

  • 正式引用前先走核验链
  • collection 结构和引文完整性交接集中在这里
  • 不把未经核验的文本直接当参考文献推进

Obsidian

Obsidian 被当成知识沉淀层，而不是和 Zotero 合并成一个工具。

  • 正式文献管理和知识综合分离
  • 项目运行产物和长期知识笔记分离
  • 更适合项目地图、方法卡和综合笔记

平台证据层

平台证据设计故意偏保守。

  • 优先浏览器可见证据
  • 证据采集和解释严格分开
  • 后续回看时仍然能检查证据 artifact

Plugin 入口层

`research-autopilot` 是公开仓库里最重要的 visitor-facing plugin anchor。

  • 入口可见
  • route explanation 可见
  • 项目升级到 squad 的路径可见

公开脚本层

公开脚本强调可迁移，不假定你的机器和我的机器完全一样。

  • 优先 repo-relative path
  • 优先环境变量
  • 优先 validator，而不是本机特判

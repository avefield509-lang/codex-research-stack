集成说明

这套系统怎么和外部工具配合。

这套公开仓库并不是“能接什么就全接上”，而是强调边界清楚： 什么地方负责正式文献，什么地方负责知识沉淀，什么地方负责证据采集。

返回中文首页 查看 GitHub

Zotero

Zotero 被当成正式文献边界，而不是一个泛化资料仓。

  • 正式引用前先走核验链
  • 文献归档和正式引用交接集中在这里
  • 不把未经核验的文本直接当参考文献推进

Obsidian

Obsidian 被当成知识沉淀层，而不是和 Zotero 合并成一个工具。

  • 正式文献管理和知识综合分离
  • 项目运行产物和长期知识笔记分离
  • 更适合项目地图、方法卡和综合笔记

平台证据层

平台证据设计故意偏保守。

  • 优先浏览器可见证据
  • 证据采集和解释严格分开
  • 后续回看时仍然能检查证据文件

插件入口层

research-autopilot 是公开仓库里最重要的用户入口。

  • 入口可见
  • 任务判断过程可见
  • 项目升级到协作模式的路径可见

公开脚本层

公开脚本强调可迁移，不假定你的机器和我的机器完全一样。

  • 优先使用仓库相对路径
  • 优先环境变量
  • 优先校验脚本，而不是本机特判

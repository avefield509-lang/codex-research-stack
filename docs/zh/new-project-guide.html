新项目上手

如果明天开一个新项目，先做什么。

这页只保留最实用的顺序：先建项目、再初始化骨架、先让系统判断任务应该怎么走，最后再决定是否需要多人协作。

返回中文首页 先看操作指南

最稳的顺序

  1. 建项目文件夹
  2. 初始化项目脚手架
  3. 先让 research-autopilot 解释适合的任务路径
  4. 如果任务变成项目型，再进入 research-team-orchestrator
  5. 用状态板和检查文件追踪当前阶段

初始化命令

pwsh -ExecutionPolicy Bypass `
  -File ".\scripts\init-research-project.ps1" `
  -Path "<PROJECT_ROOT>"

初始化后先看

  • AGENTS.md
  • research-map.md
  • findings-memory.md
  • logs/project-state/current.json

第一句怎么说

@research-autopilot 这是一个新的研究项目。
主题是：...
问题是：...
材料或数据是：...
预期产出是：...
请先解释适合的任务路径，再决定是否需要升级成项目型多智能体协作。

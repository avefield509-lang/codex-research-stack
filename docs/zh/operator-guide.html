操作指南

项目跑起来以后，你应该看什么。

这页不讲抽象概念，重点解释一件事：当系统开始工作以后，你应该从哪里判断它在做什么、是不是走偏了、接下来又该看哪一步。

返回中文首页 新项目上手

先看这条主链

  • research-autopilot：先判断这是什么任务，建议从哪里开始
  • research-team-orchestrator：只在任务已经变成项目时才介入
  • 执行技能：真正负责核验、分析、写作、导出和同步

什么才算真实多智能体

  • 独立的 agent_id
  • 独立的上下文包
  • 独立输出目录
  • 明确的审查对应关系
  • 能阻断推进的检查结果
  • 交接记录和项目状态记录

项目运行时先盯这些文件

  1. AGENTS.md
  2. research-map.md
  3. findings-memory.md
  4. material-passport.yaml
  5. logs/project-state/current.json
  6. logs/quality-gates/pipeline-status.json

为什么检查点不是装饰

这套公开仓库把正式引文、数据来源、写作质量、结果有效性、复现和最终放行都当成真正能拦住项目推进的检查点，而不是摆在那里的提醒语。

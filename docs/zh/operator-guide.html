Operator Guide

先看控制链，再看具体技能。

这页解释这套系统是什么、你通常会先看到什么、什么才算真实多智能体执行，以及项目运行时最值得盯的文件。

返回中文首页 新项目上手

公开控制链

  • `research-autopilot`：唯一顶层 router
  • `research-team-orchestrator`：只在项目型任务里进入
  • 执行 skills：真正做核验、分析、写作、导出和同步

真实多智能体的最低标准

  • 独立 `agent_id`
  • 独立 context packet
  • 独立输出目录
  • review 映射
  • target-specific gate
  • handoff / project-state trace

项目运行时最重要的文件

  1. `AGENTS.md`
  2. `research-map.md`
  3. `findings-memory.md`
  4. `material-passport.yaml`
  5. `logs/project-state/current.json`
  6. `logs/quality-gates/pipeline-status.json`

为什么 gate 不是装饰

这套公开仓库默认把 citation、data provenance、writing quality、result validity、reproducibility 和 final approval 都当作可以阻断推进的检查点。

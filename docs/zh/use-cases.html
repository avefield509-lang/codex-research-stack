典型用例

这套系统最适合哪些研究任务。

这里列的不是抽象口号，而是这套公开仓库最清楚、最稳定的几类工作模式： 文献综述、计算社会科学项目、平台证据任务，以及写作导出前的质量控制。

文献综述与正式引文边界

适合“检索一个主题、核验正式文献、组织成可继续写作的综述框架”这类任务。

  • 先进入文献综述路径
  • 任务变大后再升级成文献协作模式
  • 正式引用前强调 DOI 与元数据核验
  • 综述结果会作为项目文件保留下来，而不是只留在聊天记录里

计算社会科学项目

适合同时包含文献、平台证据、分析、写作和复现的项目型工作。

  • 进入计算社会科学任务路径
  • 初始化项目脚手架和记忆文件
  • 升级成协作式项目流程，而不是试图一口气在单线程里做完
  • 状态板、当前负责人、阻塞点和下一道检查都保持可见

平台证据与案例任务

适合从平台页面中提取浏览器真实可见内容，并把证据和后续解释拆开的任务。

  • 进入平台案例任务路径
  • 优先使用浏览器可见证据边界
  • 采集与解释分离
  • 当任务变成案例库或评论分析时，可升级为更完整的证据协作模式

写作前的导出控制

适合草稿、返修包、投稿包这类写作型工作，而不是简单格式转换。

  • 进入写作与导出路径
  • 草稿 / 返修 / 投稿包才会升级
  • 导出前要求已用引文捕获和写作质量 gate
  • 导出是核验和审查之后的环节，不是绕过它们的捷径

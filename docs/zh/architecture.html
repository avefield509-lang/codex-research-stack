Architecture

把研究工作当成一个 routed system，而不是聊天框。

这页解释 control plane、squad layer、gate layer 和 knowledge handoff 是怎么连起来的。

返回中文首页 操作指南

四层结构

  1. Control plane：`research-autopilot` 负责 route、profile 和升级判断
  2. Squad layer：`research-team-orchestrator` 负责 producer、reviewer、dispatch 和状态板
  3. Gate layer：负责 citation、writing、validity 和 reproducibility 阻断
  4. Knowledge layer：负责 Zotero、Obsidian 和项目 artifact 的交接

为什么多智能体层重要

这层的作用是防止“假多智能体”。只有 dispatch、context packet、canonical 输出目录、review mapping 和 gate log 一起出现时，它才算真实运行。

canonical artifact

  • `.codex/dispatch/`
  • `.codex/context-packets/`
  • `outputs/agent-runs/`
  • `logs/agent-handoffs/`
  • `logs/quality-gates/`
  • `logs/project-state/`

目标很简单：当一次运行失败时，人仍然能回看发生了什么。

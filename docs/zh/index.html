Codex 研究工作台

让你在 Codex 里做研究时，不再一开始就失去条理。

Codex Research Stack 解决的是研究工作最容易混乱的那一层： 任务到底该怎么开头，项目该怎么协作推进，关键检查什么时候该拦住， 最后的材料又该怎么沉淀下来。

打开 GitHub 快速开始 新项目上手
先判断任务 项目可回看 关键检查可见 材料清楚沉淀

这个仓库主要想解决什么

很多研究自动化工具在“已经知道该怎么做”之后才有用。真正容易出错的是更前面：任务判断、项目推进、检查点和材料交接。

先把任务说清楚

系统会先告诉你这是什么类型的工作、建议怎么开始，而不是直接把工具全调起来。

项目不再像长对话

项目型任务会变成一个能回看的工作区，谁在做、交给谁审、卡在哪一步都能看见。

检查点摆在台面上

引文、写作质量和复现检查会显式出现，不再靠“默认应该没问题”往前推进。

如果你第一次来看，先点这几个就够了

你不需要先把整个系统研究明白。先按你的目的选一个入口，再决定要不要继续往深处看。

我想先试一下先了解公开仓库怎么组织，再跑最基本的校验。 我想知道项目跑起来后发生什么看操作指南，理解系统会留下哪些可读痕迹。 我想直接开一个项目看新项目上手页，了解一个项目怎么从零开始。
我想先看整体结构看看任务判断、项目协作、检查和材料沉淀怎么接起来。 我想看适合什么场景浏览文献综述、平台案例、写作和计算社会科学用例。 我想看工具怎么接理解 Zotero、Obsidian、脚本和证据采集各自处在什么位置。

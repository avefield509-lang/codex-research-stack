VELA

把 VELA 放进你自己的 Codex 环境中使用，让材料、证据、主张、方法、交付物和交接记录都保留在可读的项目结构里。

开始使用 VELA 下载说明 连接 HELM HELM 仓库

VELA 提供什么

一套不依赖 app 也能运行的项目结构和工作方法。

项目结构

问题、材料、证据、主张、方法、交付物和交接都有清楚位置。

证据纪律

材料不是证据；只有来源、访问时间、核验、权利或伦理字段可见后，材料才进入证据层。

Codex 交接

交给 Codex 的任务包含文件、约束、预期输出和已知缺口。

可选看板

HELM 可以读取同一套项目状态，但 VELA 不依赖 HELM。

两个独立品牌，一套共享契约

VELA 不需要 HELM 也能安装使用。HELM 不需要 VELA 也能运行。组合使用时，两者通过显式本地文件交换状态，而不是依赖隐藏 app 状态。

VELA 到 HELM

暴露 vela.project.context.v1，让 HELM 导入项目路径、证据计数、交付物、环境健康和交接准备度。

HELM 到 VELA

导入 helm.codex.handoff.v1，作为有边界的 Codex 交接包存入项目 handoffs/ 层。

从这里开始

1. 下载

克隆或下载仓库，在你已经使用的 Codex 环境中打开。

2. 建立状态

创建项目文件夹，记录材料，只把已核验材料升级为证据。

3. 交接任务

用 VELA 的任务、文件、约束、输出和缺口格式，让 Codex 做有边界的工作。

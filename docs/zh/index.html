VELA

VELA 给 Codex 一个可携带的项目实验室：文件夹、规则、交接模板、证据结构、本地检查和 HELM 可读状态。

统一产品语言

VELA 和 HELM 是一套工作流里的两个分工，不是同一个工具。

VELA Versatile Experiment Lab & Automation

准备可携带的项目结构和有边界的 Codex 工作。

HELM Hub for Evidence, Logs & Monitoring

读取本地状态，并把它变成清楚的项目看板。

VELA 提供什么

一套不依赖桌面 app 也能运行的项目结构和工作方法。

项目实验室

vela init 创建项目文件夹、AGENTS 规则、模板、日志和机器可读状态。

证据生命周期

材料和证据分开；来源、访问、核验、权利或伦理字段可见后，材料才进入证据层。

Codex 自动化边界

交接必须说明任务、文件、约束、预期输出、复核标准和验证上下文。

HELM 连接

HELM 可以读取同一套项目状态作为看板，但 VELA 不依赖 HELM。

VELA 和 HELM 怎么协作

VELA 不需要 HELM 也能安装使用。HELM 不需要 VELA 也能运行。组合使用时，两者通过显式本地文件交换状态，而不是依赖隐藏 app 状态。

VELA → HELM

VELA 写出 vela.project.context.v1，让 HELM 展示路径、状态、证据计数、文件、阻塞和本地检查。

HELM → Codex / VELA

HELM 准备可复制的 helm.codex.handoff.v1 说明。只有用户显式保存或导出时，VELA 才应写入项目。

从这里开始

1. 安装

在克隆的仓库里运行 install.ps1 或 install.sh。

2. 初始化

运行 vela init my-project，生成可读取的本地项目上下文。

3. 交给 Codex

创建有边界任务，检查格式，渲染提示，然后回到 Codex 执行。

VELA = Versatile Experiment Lab & Automation。HELM = Hub for Evidence, Logs & Monitoring。

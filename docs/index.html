VELA Versioned Evidence Lifecycle Architecture

HELM Hub for Evidence, Logs & Monitoring

Reads local state and turns it into a clear dashboard.

What VELA Gives Codex

A portable workflow package that stays useful even when no desktop app is installed.

Project Wrapper

vela init creates project folders, AGENTS rules, Codex command templates, profiles, logs, and machine-readable state.

Evidence Lifecycle

Materials stay separate from evidence until source, access, verification, and rights or ethics fields are visible.

Schema Handoffs

Handoffs name the task, files, constraints, expected output, review standard, and human review status.

Export Checks

privacy scan and export public help keep local paths, credentials, and private material out of shared outputs.

How VELA And HELM Work Together

VELA can be installed without HELM. HELM can run without VELA. Together they exchange explicit local files rather than hidden app state.

VELA → HELM

VELA writes vela.project.context.v1 so HELM can show paths, status, evidence counts, files, blockers, and local checks.

HELM → Codex / VELA

HELM prepares helm.codex.handoff.v1 as a copyable Codex note. VELA stores it only after an explicit user save or export.

Start Here

1. Install

Run install.ps1 or install.sh in the repository you cloned.

2. Initialize

Run vela init my-project to create a project with a readable local context file.

3. Hand Off

Create a bounded task, lint it, render a prompt, and take that task back to Codex.

4. Validate

Run project validation, privacy scan, and public export checks before sharing work.

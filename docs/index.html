VELA

Versioned Evidence Lifecycle Architecture for Codex-based research work. VELA packages project structure, evidence rules, method checkpoints, deliverable flow, and handoff context so a researcher can run the workflow without installing the local board.

Getting started Install notes Integrations

Independent, but designed to pair

VELA is the workflow environment. HELM is the optional local research board. Each can stand alone; together they make project state, evidence, deliverables, environment health, and Codex handoffs easier to inspect.

VELA · 科研工作流环境

Use it inside your own Codex environment. Keep materials, evidence, claims, methods, deliverables, and handoff context in a portable workflow structure.

HELM · 本地科研看板

Use it as an independent local board when you want project status, evidence, deliverables, environment health, and handoffs in one desktop surface.

Layered

Materials, evidence, claims, methods, and artifacts stay separate until review conditions are met.

Traceable

Evidence paths and handoff decisions remain visible across the lifecycle.

Portable

The workflow is designed to move across local Codex setups without a required app dependency.

Calm

The visual system uses pale blue, white, and navy with iOS-like restraint rather than a command-console look.

Shared visual language: layered sail, evidence trace, navigation rings, and soft wave layers.

Make research work in Codex easier to follow.

Codex Research Stack helps you begin with a clearer plan, keep project work organized, and keep references, writing, and reproducibility checks in view. It is built for researchers who want more than a long chat thread.

What this repo is trying to fix

Research automation usually becomes confusing before the real work even begins. This stack makes those early decisions and handoffs easier to inspect.

Choose the right kind of work

Before tools start running, the system explains what kind of task this is and what it plans to do next.

Keep projects readable

Project work becomes a visible workspace with roles, review steps, and progress traces instead of one giant conversation.

Keep key checks visible

References, writing quality, and reproducibility stay in view as explicit checks rather than hidden assumptions.

Why this feels different from a generic agent repo

It stays inside Codex

This repo does not try to replace Codex. It makes research work inside Codex easier to organize and inspect.

It favors traceability over theater

The point is not to sound clever. The point is to leave behind files, checks, and handoffs that a human can read later.

If this helps you run research work in Codex with more clarity, open the repository and explore the public project scaffold.

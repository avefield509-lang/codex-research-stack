Codex Workflow Wrapper

VELA

Install VELA next to Codex to give each research project a bounded, evidence-aware operating layer: AGENTS instructions, handoff packets, validators, logs, and HELM-readable context.

What VELA Gives Codex

A portable wrapper package that stays useful even when no app is installed.

Starter Package

vela init creates project folders, AGENTS rules, Codex command templates, logs, and machine-readable state.

Evidence Discipline

Materials remain drafts until source, access time, verification, and rights or ethics fields are visible.

Codex Handoffs

Tasks sent to Codex include files, constraints, expected output, review standard, and validation context.

Optional Board

HELM can read the same project state, but VELA does not depend on HELM.

Two Independent Brands, One Shared Contract

VELA can be installed without HELM. HELM can run without VELA. Together they exchange explicit local files rather than hidden app state.

VELA to HELM

Expose vela.project.context.v1 so HELM can import project paths, evidence counts, deliverables, health, and handoff readiness.

HELM to VELA

HELM prepares helm.codex.handoff.v1 as a copyable Codex handoff. VELA should only store it when the user explicitly saves or exports it.

Start Here

1. Install

Run install.ps1 or install.sh in the repository you cloned.

2. Initialize

Run vela init my-project to create a VELA project with .vela/context.json.

3. Hand Off

Run vela handoff new, lint it, render a prompt, and take the bounded task back to Codex.

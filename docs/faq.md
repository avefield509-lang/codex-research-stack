# FAQ

## Is VELA an app?

No. VELA is the workflow environment package. HELM is the optional local research board.

## Do I need HELM?

No. VELA should remain usable inside a user's own Codex environment without HELM. HELM can make the same project state easier to scan.

## Is this a black-box research automation system?

No. VELA keeps project state visible. Generated suggestions are drafts until a user reviews them and the evidence fields support the claim.

## Do I need Python, Zotero, or Obsidian?

No for the base workflow. Python, Zotero, and Obsidian are optional enhancements for checks, references, and long-lived notes.

## Where should private research material live?

In the user's own project folder. Public examples and release assets must not include private notes, PDFs, browser sessions, tokens, SSH keys, or machine-specific paths.

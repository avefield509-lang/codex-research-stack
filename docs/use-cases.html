Use cases

These are the research tasks the stack handles best.

This page is not a feature list. It shows the kinds of work that fit the stack most naturally: literature review, computational social science, platform evidence, and writing or submission workflows.

Back to home View GitHub

Literature review and evidence synthesis

Useful when you need to define a review question, collect candidate literature, verify formal references, and turn the result into a usable synthesis.

Computational social science projects

Useful when literature, source collection, analysis, writing, and reproducibility need to stay tied to one visible project state.

Platform evidence and case work

Useful when you need browser-visible evidence, explicit provenance, and a clean separation between capture and later interpretation.

Writing, revision, and submission

Useful when drafts, revision packs, and final submission materials need explicit checks instead of being treated as simple export tasks.

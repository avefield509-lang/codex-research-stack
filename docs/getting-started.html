Getting started

Get the stack running before you study the whole architecture.

If this is your first visit, you do not need to learn every internal layer first. Start a demo project, see the basic structure, and only then go deeper into routes, checks, and orchestration.

Back to home View GitHub

What you will see first

  • task explanation before execution
  • a reusable project scaffold
  • visible checks and project state
  • a public-facing plugin entrypoint

Shortest safe path

  1. clone the repository
  2. read the top-level README
  3. create a demo project
  4. run the public validation scripts
  5. inspect the resulting project files

Minimal commands

git clone https://github.com/avefield509-lang/codex-research-stack.git
cd codex-research-stack
pwsh -ExecutionPolicy Bypass -File ".\scripts\init-research-project.ps1" -Path ".\examples\demo-project"
python .\scripts\validate_subagent_registry.py
python .\scripts\validate_agents_contract.py
python .\scripts\validate_research_pipeline.py
python .\scripts\validate_research_stack.py

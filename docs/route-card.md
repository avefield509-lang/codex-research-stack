# Project Path Suggestions

VELA keeps routing language close to the researcher's work. Public suggestions should point to the next inspectable action rather than to internal schema names or autonomous execution.

Typical suggestions include:

- define the research question;
- record the method expectation;
- add the first material;
- upgrade a material into evidence;
- link evidence to a claim;
- identify an evidence gap;
- prepare a bounded Codex handoff;
- export a truthful project status note.

Suggestions remain explainable and never silently mark research work as complete.

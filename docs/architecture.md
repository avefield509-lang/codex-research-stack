# Architecture

## Layers

1. `research-autopilot`
2. `research-team-orchestrator`
3. contract + gate layer
4. execution and integration layer
5. knowledge handoff layer

## Core Assets

- `skills/catalog/`
- `skills/schemas/`
- `skills/plugins/research-autopilot/`
- `scripts/`

## Design Intent

This stack does not replace Codex.
It adds a research-oriented control layer on top of Codex.

## Main Contracts

- project scope rules
- agent execution modes
- subagent registry
- reviewer allowlist
- research pipeline stages
- quality gates
- writing quality rules

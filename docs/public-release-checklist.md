# Public Release Checklist

## Public Naming

- README title uses **VELA**.
- Pages home uses **VELA** as the workflow brand.
- HELM is described as an optional, independent local research board.
- Public copy avoids older stack and automation labels as current product brands.

## Product Boundary

- VELA is understandable without installing HELM.
- HELM is not described as the controller of VELA.
- Codex handoff is described as scoped context transfer, not autonomous project execution.
- Generated suggestions are never described as verified evidence.

## Visual System

- Pale blue and white remain the dominant surface.
- Navy text, translucent layers, evidence traces, navigation rings, and soft wave layers carry the shared system.
- Text is laid out in HTML/CSS; generated images should not be relied on for final product copy.

## Public Safety

- No private notes, PDFs, browser sessions, Zotero databases, Obsidian vaults, tokens, SSH keys, account traces, or local absolute paths are included.
- Examples use placeholders such as `<PROJECT_ROOT>`, `<CODEX_HOME>`, and `<USER_HOME>`.
- Historical implementation names are marked as internal where they remain.

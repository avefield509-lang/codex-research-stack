# Workflow Router

The older automation label is no longer the public product narrative. In the VELA brand system, public copy should emphasize workflow routing, evidence state, and Codex handoff boundaries.

Public copy should describe the capability as a **workflow router** or **Codex handoff layer**:

- it turns project state into bounded next actions;
- it keeps materials, evidence, claims, methods, and deliverables separate;
- it prepares scoped context for Codex;
- it records what was requested, what changed, and what still needs review;
- it never treats generated text as verified evidence.

Implementation identifiers such as `research-autopilot` may remain as internal technical names until they can be migrated without breaking compatibility. They should not appear as the public brand.

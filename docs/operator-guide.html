Operator guide

When a project is running, these are the first things to inspect.

This page explains what the system is doing during live project work, what counts as real multi-agent execution, and which files tell you where the project stands.

Back to home Start a project

Read this control chain first

  • research-autopilot: decides what kind of task this is and where to begin
  • research-team-orchestrator: enters only when the task has become a project
  • execution skills: perform citation checks, analysis, writing, export, and sync work

What counts as real multi-agent work

  • separate agent_id values
  • separate context packets
  • separate output folders
  • explicit reviewer mapping
  • blocking checks and project-state traces

Files worth checking first

  1. AGENTS.md
  2. research-map.md
  3. findings-memory.md
  4. material-passport.yaml
  5. logs/project-state/current.json
  6. logs/quality-gates/pipeline-status.json

Why the checks matter

The stack treats reference integrity, data provenance, writing quality, result validity, and reproducibility as real checks that can stop progress. They are part of the system, not decoration.

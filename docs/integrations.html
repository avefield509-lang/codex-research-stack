Integrations

How the stack works with the tools around it.

The public stack does not connect everything just because it can. It keeps boundaries clear: formal references, long-lived knowledge notes, browser-visible evidence, and local scripts each have their own place.

Back to home View GitHub

Zotero

Zotero is the formal reference boundary. Verified formal sources move here; the rest of the workspace can still track notes and project state without pretending every source is ready for citation.

Obsidian

Obsidian is the long-lived knowledge layer. Reusable notes and synthesis can move there, while runtime logs and project checks stay inside the project itself.

Browser-visible evidence

For platform and web cases, the stack stays conservative: prefer browser-visible material, keep provenance explicit, and keep evidence capture separate from later interpretation.

Local scripts and checks

Public scripts aim to stay portable. They rely on repository-relative paths, environment variables, and visible validation instead of machine-specific assumptions.

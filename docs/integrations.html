Integrations

Optional layers, clear boundaries.

VELA can work with Codex, HELM, Python, Zotero, Obsidian, and model providers, but none of those tools should silently convert drafts into facts or make the workflow app-dependent.

Codex

Receives scoped handoff context for bounded research, code, analysis, or documentation tasks.

HELM

Reads project state as an independent local board for evidence, deliverables, health, and handoffs.

Research tools

Python, Zotero, Obsidian, and providers are optional enhancements, not prerequisites for the base workflow.

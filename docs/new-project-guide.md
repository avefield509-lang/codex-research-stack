# New Project Guide

This guide describes the VELA first-run path for a real local research project.

## 1. Define The Project

Create a portable project folder and record the research question. Keep public examples free of absolute local paths and private account details.

## 2. Capture Materials

Save source clues early: DOI, URL, file path, dataset, platform capture, policy document, field note, or manual note. Do not call a material evidence until the evidence fields are present.

## 3. Set Method Expectations

Record the method route, assumptions, data boundaries, ethics requirements, and expected deliverables. This creates the context Codex will need for bounded tasks.

## 4. Bind Evidence To Claims

A claim needs linked evidence and a support explanation. Unsupported claims can remain as candidates, but they should not be presented as findings.

## 5. Prepare Codex Handoff

Create a scoped handoff that states the task, relevant files, constraints, expected output, and known gaps. HELM can help assemble this view, but VELA can do it independently through documented project state.

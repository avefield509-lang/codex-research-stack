New project guide

If you start a new project tomorrow, do these things first.

This page keeps only the shortest practical sequence: create a project, initialize the scaffold, let the system explain the task path, and only then decide whether the project needs multi-agent coordination.

Back to home Read the operator guide

The stable sequence

  1. create a project folder
  2. initialize the project scaffold
  3. let research-autopilot explain the task path first
  4. if the work becomes project-type, let research-team-orchestrator take over
  5. track progress through project state and checks

Initialization command

pwsh -ExecutionPolicy Bypass `
  -File ".\scripts\init-research-project.ps1" `
  -Path "<PROJECT_ROOT>"

Look at these files first

  • AGENTS.md
  • research-map.md
  • findings-memory.md
  • logs/project-state/current.json

A good first prompt

@research-autopilot This is a new research project.
The topic is: ...
The question is: ...
The materials or data are: ...
The expected outputs are: ...
Please explain the most suitable task path first, then decide whether the project should upgrade into multi-agent coordination.

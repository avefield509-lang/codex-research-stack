# AGENTS.md

## VELA Contract

You are operating inside a VELA workflow wrapper package.

## Core Boundaries

- Do not merge materials, evidence, claims, methods, deliverables, and handoffs.
- Do not treat collected material as verified evidence.
- Do not present a claim as supported unless it links to evidence.
- Do not expose private paths, credentials, local machine details, or personal notes.
- Do not rewrite the whole project unless the handoff explicitly asks for it.

## Before Acting

1. Read the current handoff packet.
2. Identify only the files relevant to that handoff.
3. State assumptions if the handoff is underspecified.
4. Prefer small edits over broad rewrites.

## Definition of Done

A task is complete only when:

- the expected output is produced;
- changed files are listed;
- validation commands are run or skipped with a reason;
- unsupported claims or privacy risks are reported.

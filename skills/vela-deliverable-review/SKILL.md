# VELA Deliverable Review

Use this skill before a report, brief, figure, table, or export leaves the project.

Check for:

- unsupported claims;
- missing evidence links;
- private paths or local usernames;
- credentials or tokens;
- restricted notes or unredacted source material;
- validation commands that were skipped without reason.

Report blockers separately from warnings. Do not rewrite the deliverable unless the handoff explicitly asks for edits.
